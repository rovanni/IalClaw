O IalClaw possui uma experiência simples, pensada para conversa direta, e uma experiência avançada, voltada para acompanhamento técnico, traces, observabilidade e controle do comportamento do executor.

Modo simples

Converse sem complicação. Peça o que você precisa e o IalClaw responde de forma direta. Essa tela foi pensada para quem só quer conversar com o agente, pedir uma tarefa ou tirar uma dúvida, sem precisar acompanhar grafo, traces ou logs internos.

  • Bom para começar: peça algo simples, como criar um jogo, corrigir um texto ou explicar um erro.
  • Recomendado para usuários leigos e uso cotidiano.
  • Também inclui histórico de conversas e opção de começar um novo chat.

Modo avançado

Use esta tela quando quiser acompanhar o raciocínio do agente, eventos de debug, histórico técnico, replay de traces, o grafo cognitivo em tempo real e trocar o modo operacional do executor.

  • Indicado para desenvolvimento, suporte e troubleshooting.
  • Mostra telemetria, repair loop e eventos internos do executor.
  • Ideal para investigar falhas, validar comportamento e ajustar o nível de risco da execução.

Modos de execução

Na tela avançada existe um seletor chamado MODO no topo. Ele controla como o IalClaw trata edição de arquivos, fallback e validação estrutural durante a execução.

Seguro

Use quando quer estabilidade

Esse modo é o mais conservador. Ele prioriza segurança durante alterações em arquivos existentes.

  • Exige diff confiável antes de editar arquivos já existentes.
  • Se o patch falhar, o overwrite completo é bloqueado.
  • Ideal para mudanças delicadas, correções cirúrgicas e revisão técnica.
Equilibrado

Use como padrão

Esse modo busca concluir a tarefa sem abandonar boas proteções. É o comportamento padrão recomendado.

  • Tenta aplicar diff primeiro.
  • Se necessário, faz fallback inteligente para overwrite.
  • Validação estrutural é leve para preservar progresso.
Livre

Use para prototipar rápido

Esse modo prioriza resultado imediato. Ele reduz barreiras de execução e aceita mais risco.

  • Pode sobrescrever diretamente quando isso acelera a entrega.
  • Pula a validação estrutural do projeto no executor.
  • Ideal para experimentação, testes rápidos e primeiras versões.
1

Abra a tela adequada

Entre em /simple para uma conversa limpa ou em /advanced para o painel técnico completo.

2

Escolha o modo de execução

No painel avançado, use o seletor MODO no topo. Escolha Seguro para estabilidade, Equilibrado para uso padrão ou Livre para prototipação rápida.

3

Escreva um pedido claro

Exemplos: “Crie uma landing page para meu produto”, “Explique por que meu projeto falhou” ou “Crie um jogo da cobrinha em HTML”.

4

Use o histórico quando quiser continuar

Tanto no modo simples quanto no avançado você pode abrir conversas salvas e continuar o contexto de uma sessão anterior.

5

Troque para o avançado quando precisar investigar

Se aparecer um comportamento estranho ou você quiser entender o fluxo interno do agente, mude para a tela avançada, acompanhe os traces e veja qual modo estava ativo durante a execução.

6

Interprete o resultado pelo contexto do modo

Se uma tarefa não concluir no modo Seguro, isso pode ser uma proteção deliberada. Se uma tarefa concluir no modo Livre, confira o resultado final com mais atenção, porque o sistema priorizou velocidade.
