# KB-049.1 - Fase de Execucao Funcional (Aplicacao dos Fixes Reais)

Data: 2026-04-07
Status: proposto
Dependencia: plano estrutural concluido em `docs/architecture/plans/KB-049.1-PLANO.md`

---

## Contexto

Esta fase aplica comportamento real apos a etapa estrutural.

Objetivo:

1. Garantir bypass real de loop para SMALL_TALK.
2. Garantir prompt/contexto social limpo.
3. Corrigir inconsistencia de MEMORY_QUERY implicito curto.
4. Humanizar capability gap sem leak tecnico.

Regra de seguranca desta fase:

1. Reutilizar logica existente antes de criar qualquer bloco novo.
2. Nao duplicar fluxo entre Controller, Loop e Orchestrator.
3. Preservar safe mode em todos os gates aplicaveis.

---

## Escopo da Implementacao

Implementar apenas:

1. Bypass real para caminho `skipPlanning=true` com corte efetivo de entrada no loop.
2. Modo social no prompt para SMALL_TALK.
3. Cobertura de MEMORY_QUERY soft para follow-up implicito.
4. Mapeamento amigavel de capability gap no output final.

Fora de escopo:

1. Refatoracoes amplas fora dos pontos do KB-049.1.
2. Mudanca de heuristicas nao relacionadas aos 4 problemas.
3. Reescrita de arquitetura de signals.

## 🧩 ESCOPO DA IMPLEMENTACAO

👉 Implementar:

FASE KB-049.1-E1 - Bypass Real de Small Talk

### E1.1 - AgentController: gate de bypass real

Arquivo:

1. `src/core/AgentController.ts`

Acoes:

1. Identificar o ponto em que `orchestrationResult` ja esta disponivel.
2. Reutilizar caminho de execucao direta existente (nao criar fluxo paralelo novo).
3. Se `skipPlanning === true`, retornar imediatamente pelo caminho direto.
4. Adicionar log tecnico de auditoria para o bypass.

Criterio de aceite:

1. Entrada SMALL_TALK nao chama `AgentLoop.run(...)`.
2. Nao aparece `minimal_plan_created` nesse cenario.

### E1.2 - AgentLoop: defesa em profundidade

Arquivo:

1. `src/engine/AgentLoop.ts`

Acoes:

1. Adicionar guard de protecao caso loop seja chamado indevidamente com flags de bypass.
2. Reutilizar sinais ja existentes; nao criar nova heuristica decisoria.
3. Manter safe mode (`orchestratorDecision ?? loopDecision`).

Criterio de aceite:

1. Sem emissao de `tool_selection_active_decision` para SMALL_TALK em caminho bypass.

## Regras Arquiteturais e de Autoridade

1. Orchestrator continua como decisor principal.
2. AgentLoop executa e aplica safe mode quando aplicavel.
3. Nao introduzir mini-brain local novo.
4. Nao remover branches existentes sem validacao de impacto.

Safe mode obrigatorio:

```ts
finalDecision = orchestratorDecision ?? loopDecision
```
